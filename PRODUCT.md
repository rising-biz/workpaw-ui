# Product

## Register

product

## Users

WorkPaw 是企业级私有化、多用户的 AI 智能体平台，基于 QwenPaw v1.1.12 定制。每个企业用户拥有独立隔离的 QwenPaw 容器实例（独立 Pod + Ingress 域名 + Access Token），通过三条路径使用：

- **workpaw-desktop（Tauri 2 桌面客户端）** — 企业员工的日常入口。SSO 登录后，管理专属容器的启动/停止，进行 Chat 对话与历史 Sessions 管理。高频、沉浸式场景：在桌面工作流中随时唤起智能体。
- **workpaw-web（Web 配置应用）** — 同一批企业员工的配置面。登录后直连 QwenPaw Pod，配置 Agent、技能、模型 Provider、渠道（钉钉/飞书/微信等）、安全策略、备份、环境变量。低频但高 stakes：配错一次，智能体就跑不动。
- **workpaw-admin/console（管理后台）** — 平台管理员。治理全部用户实例的生命周期、资源用量、审计日志、全局策略、OIDC 连接。

三类用户共享同一套品牌与同一份信任契约：数据私有、操作可审计、界面专业可靠。

## Product Purpose

把 QwenPaw（面向个人的开源智能体）改造成企业可部署、多用户隔离、可治理的私有化平台。后端沿用 QwenPaw 容器（锁定 v1.1.12），定制三件事：前端交互界面、企业级 SSO/权限/审计、K8s 实例编排。

为什么存在：企业想用 AI 智能体，但不能接受公有云 SaaS 的数据出境和共享租户。WorkPaw 让每家企业把 QwenPaw 跑在自己的 K8s 集群里，每个员工一个独立 Pod，数据隔离在各自的 PVC 里，全部操作留痕可审计。

成功标准：员工 5 分钟内完成 SSO 登录、启动容器、正常对话（流式输出 + 文件上传）；管理员一眼看到全平台实例状态、资源用量和操作轨迹；配置面不吓人，运维面不冷硬，日常对话面有温度。

## Brand Personality

**轻盈、明快、友好**（Light · Bright · Friendly）

- **轻盈（Light）** — 不压人。充足的留白、克制的信息密度、微暖的底色像一本好书的纸。企业软件不必沉重；它可以是工具，不是负担。
- **明快（Bright）** — 有活力但不聒噪。靛蓝品牌色提供清晰的视觉信号，不沉闷、不刺眼。界面让人愿意打开，而不是不得不用。
- **友好（Friendly）** — 让人感到被照顾。微交互有回应，错误有解释，空状态有引导。不冷漠、不官僚、不像一个监视系统。

情绪目标：员工打开 desktop 时感到"这是我的 AI 助手，随时帮我"；配置者打开 web 时感到"我搞得定"；管理员打开 admin 时感到"一切尽在掌控"。三个界面共享同一种亲和力，让企业软件不吓人。

品牌色：靛蓝（Indigo）作为默认主题锚点。三主题体系支持用户切换——靛蓝工作室（默认）、暖光桌面、天空画布——覆盖从专注到温暖到通透的不同偏好，全部保持"轻盈·明快·友好"的基因。

## Anti-references

明确不想长成的样子：

- **QwenPaw 的 Ant Design 观感** — 功能照搬，气质重塑。不用 AntD 紧凑表单密度、默认配色、组件间距节奏。WorkPaw 用 Shadcn UI + base-ui。
- **冷硬企业控制台** — 不像 AWS Console / Grafana 那样密集低对比、黑底荧光、只有工程师能看懂。admin 也是给人用的。
- **过度装饰的 SaaS** — 不要毛玻璃卡片、渐变文字（background-clip:text + gradient）、32px+ 超大圆角卡片、营销式 Hero 模板。
- **沉闷的企业蓝** — 不以"安全"为名选择沉闷的深蓝/灰色调。蓝色在企业软件中已经饱和，WorkPaw 的靛蓝在蓝紫之间找到自己的区间。
- **AI-SaaS 暖中性底色** — 不向 2026 年饱和的米色/沙色/奶油色默认倾斜。靛蓝主题的微暖白是"纸色"（chroma 0.003），不是"米色"（chroma 0.01+）。
- **等大卡片网格** — 不用千篇一律的 icon + 标题 + 正文等大卡片。信息架构决定容器，而非反向。

## Design Principles

1. **工具感优先，不装饰** — 界面的第一身份是工具。每个元素都为功能服务，装饰只在它真正改善体验时存在。留白本身就是设计。
2. **三界面一套骨架** — desktop、web、admin 共享同一套 token、组件语义、排版节奏。跨界面切换不感到换了产品。一致性 = 信任。
3. **信号稀缺，信息丰富** — 品牌色占任何屏幕 ≤10%，只在主操作、聚焦态、关键状态点出现。它的稀缺是它被看见的前提。信息密度由层级和留白控制，不由颜色控制。
4. **对所有人都清晰** — WCAG AA 是硬约束。正文对比 ≥4.5:1、键盘可达、读屏完整、色盲安全图表。中文第一版，i18next 框架保留以备未来。
5. **选择权给用户** — 三主题（靛蓝、琥珀、天蓝）让用户选自己喜欢的底色和强调色，而非强加一种审美。主题切换即时生效，跨界面同步。

## Accessibility & Inclusion

**WCAG 2.1 AA，严格达标。** 三个前端界面均适用。

- **对比度**：正文 ≥4.5:1，大字（≥18px 或 bold ≥14px）≥3:1。muted-foreground 在各主题亮色模式下对比 ≥5:1，暗色模式下 ≥4.5:1。占位符文本同样 4.5:1，不用默认浅灰。
- **键盘可达**：所有交互完整键盘可达，可见焦点环（品牌色 ring，3px）。
- **读屏**：语义化 HTML + ARIA 标签，图标按钮有 aria-label，表单控件有关联 label。
- **减弱动效**：每个动画都有 `@media (prefers-reduced-motion: reduce)` 的降级（交叉淡入或瞬切）。
- **色盲安全**：状态指示不只靠颜色（配图标/文字），图表用 5 色色盲安全调色板。
- **中文第一版**：界面文案仅中文（i18next 框架保留），排版兼顾中文阅读体验（字距、行高、标点）。
