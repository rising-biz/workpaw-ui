# Product

## Register

product

## Users

WorkPaw 是企业级私有化、多用户的 AI 智能体平台,基于 QwenPaw v1.1.12 定制。每个企业用户拥有独立隔离的 QwenPaw 容器实例(独立 Pod + Ingress 域名 + Access Token),通过三条路径使用:

- **workpaw-desktop(Tauri 2 桌面客户端)** — 企业员工的日常入口。SSO 登录后,管理自己专属容器的启动/停止,进行 Chat 对话与历史会话(Sessions)管理。这是高频、沉浸式的使用场景:在桌面工作流中随时唤起一个能用的智能体。
- **workpaw-web(Web 配置应用)** — 同一批企业员工的配置面。登录后直连自己的 QwenPaw Pod,配置 Agent、技能、模型 Provider、渠道(钉钉/飞书/微信等)、安全策略、备份、环境变量。低频但高 stakes:配错一次,智能体就跑不动。
- **workpaw-control-plane/console(管理后台)** — 平台管理员。治理全部用户实例的生命周期、资源用量、审计日志、全局策略、OIDC 连接。面向的是对合规和可观测性负责的运维/安全角色。(原独立仓库 workpaw-admin,已并入 workpaw-control-plane 的 console/ 子目录,前后端分离部署)

三类用户共享同一套品牌与同一份信任契约:数据私有、操作可审计、界面专业可靠。首要工作流是"配置好 → 用起来":web/admin 把智能体配好,desktop 把它用起来。

## Product Purpose

把 QwenPaw(一个面向个人的开源智能体)改造成企业可部署、多用户隔离、可治理的私有化平台。后端完全沿用 QwenPaw 容器(锁定 v1.1.12),只定制三件事:前端交互界面、企业级 SSO/权限/审计、K8s 实例编排。

为什么存在:企业想用 AI 智能体,但不能接受公有云 SaaS 的数据出境和共享租户。WorkPaw 让每家企业把 QwenPaw 跑在自己的 K8s 集群里,每个员工一个独立 Pod,数据隔离在各自的 PVC 里,全部操作留痕可审计。

成功是什么:一个企业员工能在 5 分钟内完成 SSO 登录、启动自己的容器、和智能体正常对话(流式输出 + 文件上传);一个管理员能一眼看到全平台的实例状态、资源用量和谁在什么时候做了什么。配置面不吓人,运维面不冷硬,日常对话面有温度。

## Brand Personality

**精密、温暖、克制** (Precise · Warm · Restrained)

- **精密 (Precise)** — 工程感的骨架:一致的网格、清晰的字号层级、精确的对齐、可信的信息密度。这是企业软件该有的"被认真做过"的质感,不是随手拼的模板。
- **温暖 (Warm)** — 人味的表面:克制的微交互、被恰当使用的品牌绿、像平易近人的助手而非冷冰冰的控制台。让人愿意用,而不是不得不用。
- **克制 (Restrained)** — 不靠装饰建立信任。没有毛玻璃、没有渐变文字、没有营销式 Hero 模板。每一个元素都为功能服务,留白本身也是设计。

情绪目标:员工打开 desktop 时感到"这是一个帮我的工具,不是一个监视我的系统";管理员打开 admin 时感到"我知道现在发生了什么,而且我能掌控它";配置者打开 web 时感到"我能看懂每一项配置的含义"。

品牌主色锁定 Supabase 绿 `oklch(0.78 0.16 162)`(#3ECF8E,沿用 Supabase 的开发者工具身份锚点)。这是温度感的载体,但用得克制——主要出现在主操作按钮、关键强调和状态指示上,而非铺满整个界面。原 QwenPaw 的 #FF7F16 橙色已退出品牌体系,仅保留为图表辅色(chart-3)以保 5 色色盲安全。

## Anti-references

明确不想长成的样子:

- **QwenPaw 的 Ant Design 观感** — 我们沿用 QwenPaw 的功能,但 UI 必须摆脱 AntD 的视觉语言(紧凑的表单密度、AntD 默认配色、组件间距节奏)。WorkPaw 用 Shadcn UI + base-ui,追求更扁平、更克制、节奏不同的企业级观感。
- **泛 AI-SaaS 米色** — 不要 2026 年饱和的暖中性底色(米色/沙色/奶油色背景)、不要千篇一律的等大卡片网格 + 图标标题正文模板。底色保持真正的中性白(或向品牌色相微微偏一点),不向暖色默认倾斜。
- **冷硬运维终端** — admin 不是 Grafana 式密集低对比的纯数据后台。保持可亲、有层级、能读得下去,不要黑底荧光绿那种"只有工程师能看懂"的观感。
- **时髦玻璃/渐变 SaaS** — 应用界面不要毛玻璃卡片、渐变文字、超大圆角(32px+)卡片、营销式 Hero 大图模板。这些是企业工具的噪音。

## Design Principles

1. **工程感的骨架,温度感的表面** — 结构层面追求精密与一致(网格、层级、对齐、密度都经得起推敲),细节层面注入人味(微交互、得体的强调色、不冷漠的文案)。精密是地基,温度是体验,两者不冲突。
2. **克制即信任** — 企业用户对"花哨"天然警惕。每多一处装饰,就少一分信任。默认做减法:能用边框就不用阴影,能用间距就不用卡片,能用文字就不用图标。装饰只在它真正改善体验时出现。
3. **功能照搬,气质重塑** — 忠实复刻 QwenPaw 的功能与信息架构(页面、字段、状态机),但视觉身份必须是 WorkPaw 自己的企业语言。用户从 QwenPaw 迁移过来时,功能都在、逻辑不变,但一眼能看出"这是企业版"。
4. **一套语言,三个界面** — desktop、web、admin 共享同一套设计 token、同一套组件语义、同一套排版与间距节奏。跨界面切换时,用户不该感到换了产品。一致性本身就是产品力。
5. **对所有人都清晰** — WCAG AA 是硬约束,不是 nice-to-have。正文对比 ≥4.5:1、大字 ≥3:1、全键盘可达、读屏标签完整、减弱动效支持、色盲安全图表。企业部署有真实合规义务,清晰性是承诺。

## Accessibility & Inclusion

**WCAG 2.1 AA,严格达标。** 三个前端界面均适用,不因界面角色而放松。

- **对比度**:正文 ≥4.5:1,大字(≥18px 或 bold ≥14px)≥3:1。muted-foreground 当前 `oklch(0.556 0 0)` 在中性白底上对比约 4.6:1,边界值,需复核每处实际使用;暗色模式下 `oklch(0.708 0 0)` 同样需复核。占位符文本同样要 4.5:1,不用默认浅灰。
- **键盘可达**:所有交互(导航、选择器、对话框、表格、Agent 切换)完整键盘可达,可见焦点环。
- **读屏**:语义化 HTML + ARIA 标签,图标按钮有 aria-label,表单控件有关联 label。
- **减弱动效**:每个动画都有 `@media (prefers-reduced-motion: reduce)` 的降级(交叉淡入或瞬切)。
- **色盲安全**:状态指示不只靠颜色(配图标/文字),图表用色盲安全调色板。
- **中文第一版**:界面文案仅中文(i18next 框架保留以备未来),但排版与组件需兼顾中文阅读体验(字距、行高、标点)。
