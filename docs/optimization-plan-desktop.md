# WorkPaw Desktop 终端用户交互优化方案

> 基于: QwenPaw 源码分析 (v1.1.12, 38+ API 模块, 25+ 前端页面) + AI Agent 竞品调研 (2026.07)
> 定位: Desktop = Tauri 原生桌面客户端，面向终端用户的 Agent 交互界面
> 当前范围 (per memory): Chat + Sessions + 文件管理器 + 场景画廊

---

## 一、QwenPaw 能力全景 vs WorkPaw Desktop 现状

### 1.1 QwenPaw Console 已有功能（25+ 页面）

| 功能域 | QwenPaw 页面 | WorkPaw Desktop 现状 |
|--------|-------------|---------------------|
| **Chat** | 多 Agent 对话、消息队列、Slash Commands、审批卡片、语音输入、Loop 模式 | ✅ 已有 Chat（基础） |
| **Inbox** | 审批中心 + 推送消息 + Harvest | ❌ 未实现 |
| **Workspace** | Agent 文件浏览器/编辑器 | ⚠️ 部分（文件管理器，但风格不同） |
| **Sessions** | 会话监控 + 会话抽屉 | ✅ 已有 Sessions |
| **Cron Jobs** | 定时任务管理 | ❌ 未实现 |
| **Heartbeat** | 心跳监控 | ❌ 未实现 |
| **Agent Config** | Agent 设置（模型/重试/限流/工具级别） | ❌ 未实现 |
| **Agent Stats** | 单 Agent 统计仪表盘 | ❌ 未实现 |

### 1.2 WorkPaw Desktop 独有的优势（QwenPaw 没有的）

| 能力 | 说明 |
|------|------|
| **SSO 深度链接** | `workpaw://` 协议回调，系统浏览器 → 桌面 App |
| **Linux 风格文件管理器** | 双栏（树 + 列表）、9 种格式预览、生命周期管理 |
| **场景画廊（做同款）** | 模板驱动的 Agent 快速创建 |
| **企业策略沙箱** | ConfigReconciler + template_bindings，声明式配置管理 |
| **K8s Pod 管理** | 通过 Control Plane + Operator 管理 Agent Pod 生命周期 |
| **桌面原生性能** | Tauri 原生窗口、本地文件系统访问、系统通知 |

## 二、Desktop 优化总览

```
QwenPaw 已有              WorkPaw 继承              WorkPaw 提升
─────────────────────────────────────────────────────────────────
Chat (基础)        →     Chat (保留)          →   企业级 Chat（审批流、策略可见性）
Inbox              →     Inbox (新增)         →   统一通知中心
Sessions           →     Sessions (增强)       →   会话审计 + 回放
Cron Jobs          →     定时任务 (新增)       →   企业调度策略
Agent Config       →     Agent 设置 (新增)     →   策略可视化
Heartbeat          →     健康监控 (新增)       →   集群级 Pod 监控
```

> **已砍掉**: DAG 工作流编排（用户是员工不是 Agent 开发者）。Coding Mode（开发者有专业 IDE，不需要 WorkPaw 再造一个）。

## 三、分阶段优化方案

### 阶段一 (P0): 核心交互升级 — 打造"精密控制台"体验

#### 3.1 统一通知中心 (Inbox)

**QwenPaw 现状**: Web Inbox（审批卡片 + 推送消息 + Harvest 杂志）

**WorkPaw 优化**:
- **系统原生通知**: Tauri 桌面通知（审批请求实时弹窗，支持快捷操作）
- **通知分级**: 
  - 🔴 紧急: Agent 危险操作审批（如 `rm -rf`）
  - 🟡 重要: Cron 任务失败、Pod 异常
  - 🟢 普通: Token 用量提醒、Agent 完成任务
- **Dock/Tray 角标**: 未读审批数在 Dock 图标显示
- **审批快操作**: 通知气泡直接 Approve/Deny，无需切到 App 内
- **操作回放**: 每项审批关联操作上下文（Agent 当时在做什么、影响了哪些文件），事后可回放

#### 3.2 Agent 交互增强

**消息队列增强**（QwenPaw 已有基础）:
- 支持多消息排队发送，后台持续运行，系统托盘显示进度
- 消息模板: 常用 prompt 保存为模板，支持参数占位符

**Slash Commands 增强**:
- 桌面端全局快捷键唤起命令面板（类 VS Code Command Palette）
- 支持自定义企业级命令（如 `/deploy`、`/audit`）

**上下文可见性**:
- 显示 Agent 当前上下文使用率（token 用量进度条）
- 显示 Agent 连接的 MCP 工具集、启用的 Skills
- 显示当前策略沙箱状态（哪些操作被限制、什么级别需要审批）

---

### 阶段二 (P1): 自主运行与定时任务

#### 4.1 Cron 定时任务

**QwenPaw 现状**: 基础 Cron 管理 API（APScheduler，支持 cron/once 两种调度类型）

**WorkPaw 增强**:
- **Cron 管理界面**: 在 Desktop 中直接管理自己的定时任务（创建/编辑/暂停/删除），无需切换到 Web 端
- **企业调度策略**: 支持工作日/非工作日、业务时段、错峰执行
- **失败处理**: 重试策略 + 通知（任务失败时系统通知 + 托盘提醒）
- **执行日历视图**: 可视化展示未来 7 天的定时任务安排
- **最近执行历史**: 展示最近 N 次执行结果和日志摘要

#### 4.2 后台运行模式

**QwenPaw 现状**: Agent 在 Web console 关闭后无法继续执行

**WorkPaw 优化**（桌面端独有优势）:
- Agent 长时间任务可最小化到系统托盘，后台持续运行
- 托盘图标动画指示 Agent 状态（思考中/等待审批/完成/异常）
- 右键托盘快捷操作: 查看进度、暂停、终止
- 任务完成时系统通知弹窗提醒

---

### 阶段三 (P2): 企业级差异化体验

#### 5.1 策略沙箱可视化

**价值**: 让终端用户感知企业安全策略，而非黑盒限制

- **策略可见性面板**: 显示当前用户/Agent/工具的权限边界
  - 🟢 允许: 读文件、搜索代码、调用 API
  - 🟡 需审批: 写文件、执行 Shell、访问网络
  - 🔴 禁止: 访问 `/etc/passwd`、修改系统配置
- **操作前提示**: 在 Chat 输入框中实时检查即将执行的操作是否需要审批
- **审批历史**: 个人的审批记录，可回放每次审批的操作上下文

#### 5.2 企业协作特性

- **Agent 共享**: 团队成员间共享 Agent 配置和对话上下文
- **Session 共享**: 将当前 Session 分享给同事继续（带权限控制）
- **实时协作**: 多用户同时与同一 Agent 对话（类似共享画布）
- **@提及**: Chat 中 @同事，将对话片段转发给对应人员

#### 5.3 场景画廊增强

**QwenPaw 现状**: 无此功能

**WorkPaw 已有**: 场景画廊 + 做同款

**优化方向**:
- **企业模板库**: 管理员推送官方模板到全员的场景画廊
- **一键部署**: "做同款"不仅复制 prompt，还包括工具配置、MCP 连接、Cron 设置
- **场景评分/评论**: 员工对模板打分和评论，促进内部知识共享
- **场景推荐**: 基于用户角色和使用习惯推荐场景

#### 5.4 桌面端独有交互

- **全局快捷键**:
  - `Cmd+Shift+A`: 唤起 Agent 对话窗口（类 Spotlight）
  - `Cmd+Shift+T`: 快速创建任务
  - `Cmd+Shift+N`: 查看通知中心
- **拖拽操作**:
  - 文件拖入 Chat 自动上传到 Agent workspace
  - 图片拖入 Chat 自动作为多模态输入
- **截图工具**: 截图 → 自动发送给 Agent（类 Cline 的截图分析）
- **剪贴板监控**: 复制代码片段 → Agent 自动问 "是否要分析这段代码？"

## 四、导航结构建议

### 当前 Desktop 导航

```
Chat | Sessions | 文件管理 | 场景画廊
```

### 建议优化后导航

```
 主区域                    左侧栏
┌──────────────────────┐
│ 🏠 Home (概览仪表盘)  │  📋 最近会话
│ 💬 Chat              │  📁 最近文件
│ ⏰ 定时任务           │  🔔 待审批 (N)
│ 🖼 场景画廊           │
│ 📁 文件管理           │
│ ⚙️ 设置              │
└──────────────────────┘
```

### Home 仪表盘（新增）

概览今日 Agent 使用情况:
- 今日对话数 / Token 消耗
- 运行中的长时间任务
- 待审批操作
- 即将执行的定时任务
- Agent 健康状态（Pod 运行状态）

## 五、与 WorkPaw Web 的分工

| 功能 | Desktop | Web |
|------|---------|-----|
| Chat | ✅ 主要交互界面 | ✅ 备用 |

| 场景画廊 | ✅ 做同款 | ❌ |
| 定时任务管理 | ✅ 个人管理 | ❌ |
| Agent 配置 | 基础设置 | ✅ 完整管理 |
| Skills 管理 | ❌ | ✅ |
| MCP 管理 | ❌ | ✅ |
| Models 管理 | ❌ | ✅ |
| Channels 配置 | ❌ | ✅ |

**原则**: Desktop = 交互 + 执行 + 消费；Web = 配置 + 管理 + 治理

## 七、关键指标

- **日活用户 Agent 交互次数**: Chat 会话数 + 消息量
- **桌面通知响应率**: 审批通知触达后用户在 5 分钟内响应的比例
- **场景画廊复用率**: 从模板创建的场景占比
- **Agent 后台运行时长**: 用户关闭窗口后 Agent 继续运行的平均时长
- **审批处理效率**: 从通知触达到完成审批的中位时间
